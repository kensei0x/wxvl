### 🎯 一、资产挖掘：备案反查与“过期复活”  
- **备案号逆向**  
：通过天眼查/爱站/ICP备案号扒出所有关联域名。重点盯着 2021-2022  
 年的旧域名，很多企业**域名过期没续费，但服务器IP还在跑旧服务**  
（如 old.xxx.com  
、static.xxx.com  
、ftp.xxx.com  
），那是未授权和旧站后台的乐园。  
  
- **404 页面反向工程**  
：随便访问不存在路径触发 404，看返回头（Server字段）、页面底部或JS加载。常会泄露**真实后端IP、内部网关、第三方微服务路由**  
。顺着特征拼接口，一找一个准。  
  
- **“兄弟公司”资产链**  
：大厂的全资子公司、收购品牌、投资公司安全基本“裸奔”。只要规则允许，直接冲子公司的 **SpringBoot Actuator**  
、Git泄露或同C段旁注（8080/9000等端口）。  
  
- **证书透明度（CT）挖掘**  
：使用 crt.sh  
 反查 %.target.com  
。很多时候企业备案了 A 域名，但在证书 SANs 列表里泄露了未备案的 B 内部域名（如 internal-dev-auth.target.com  
）。  
  
### 🎯 二、降维打击：接口版本号与生态盲区  
- **版本号“倒行逆施”**  
：看到 /api/v3/user  
 鉴权很严，立刻试 v2/v1/v0  
 或无版本号路径。新版修了漏洞，**旧版接口往往为了兼容而保留，且几乎不校验 Token**  
。  
  
- **微信/钉钉/小程序生态**  
：重点看小程序开发者工具抓包 wx.request  
。大量小程序后台**无鉴权、存在越权**  
，直接返回用户列表/身份证。另外留意 /api/wx/getUserInfo  
 等接口获取 OpenID 或手机号。  
  
- **APP 离线包/热更新挖洞**  
：用 apktool  
 解压 APK 搜 index.android.bundle  
、patch.zip  
 或 assets  
 目录。里面常泄露**测试后台、写死的 Token/Secret、后台账号密码**  
。  
  
### 🎯 三、鉴权绕过与冷门入口（原创技巧）  
- **参数名越权**  
：userId 失败就试 id/uid/userid/account  
。后端有时只校验参数名是否匹配，不校验值归属，且 key 对应的权限逻辑可能不同。  
  
- **路径与 Cookie 绕过**  
：尝试路径大小写 /Admin  
、双斜杠 //admin  
。或将 Cookie 中的 role=user  
 改为 admin  
，isLogin=false  
 手动改为 true  
，欺骗低级前端逻辑。  
  
- **短信/邮件藏宝图**  
：注册时收到的验证短信/链接中常包含 verify.xxx.com  
、task.xxx.com  
 等无人扫描的隐蔽域名，高产**任意用户验证**  
漏洞。  
  
- **Swagger/Knife4j 入口**  
：看到 /swagger-ui.html  
、/doc.html  
 别只看文档，直接点“调试”。90% 的接口无需登录即可调用，带 admin  
 字眼的接口优先。

### 🎯 六、老兵独家：1分钟速刷模板（不出洞直接跑）  
  
**核心逻辑：按照“权限 -> 泄露 -> 绕过 -> 资产”的顺序，5步闭环。**  
#### Step 1. 权限孤岛测试（判定 P1 级未授权）  
- **操作**  
：打开核心业务页（订单/用户/支付），F12 抓包，**彻底删除**  
 Header 中的 Cookie  
、Authorization  
、Token  
。  
  
- **判定**  
：回显 200 且 JSON 包含敏感字段 -> **直接定级 P1**  
；回显 401/403 -> 进入 Step 2。  
  
#### Step 2. 核心端点碰撞（判定 P1/P2 级配置泄露）  
- **操作**  
：根目录直接拼接：/actuator/heapdump  
、/actuator/env  
、/nacos  
、/swagger-ui.html  
。  
  
- **判定**  
：heapdump  
 下载成功 -> **P1 斩杀线**  
（本地提取库密）；其他端点成功 -> P2/P3。  
  
#### Step 3. 语义差异绕过（判定权限绕过漏洞）  
- **操作**  
：对 Step 1 中失败的接口尝试变形：/api/v1/user  
 改为 /api/v1/user/..;/  
 或 /api/./v1/user  
。  
  
- **判定**  
：绕过 WAF 成功读取数据 -> P2 级越权。  
  
#### Step 4. 源码深度挖掘（判定信息泄露）  
- **操作**  
：F12 -> Sources -> Ctrl+Shift+F 全局搜：LTAI  
 (阿里云)、eyJ  
 (JWT)、10.  
 (内网IP)、.map  
 (SourceMap)。  
  
- **判定**  
：发现云密钥 -> **P1 严重**  
；发现内网拓扑 -> 辅助渗透。  
  
#### Step 5. 影子资产关联（判定边缘弱口令）  
- **操作**  
：看短信/邮件中的域名（如 test-link.xxx.com  
），或同 IP 偏移测试（8080/9000）。  
  
- **判定**  
：发现默认口令 admin/admin  
 或测试环境入口 -> 稳收 P2。  
  
> ⚠️ **老兵箴言**  
：1 分钟走完这 5 步，不出洞立刻关网页，绝不浪费时间。SRC 刷分比的是“广度 × 深度”。

## 📄 SRC 高分漏洞报告模板（通用可直接提交）  
> ⚠️ **报告原则**  
：清晰 + 可复现 + 危害讲透 + 脱敏合规，不要多余废话  
  
### 一、漏洞基础信息  
```
漏洞标题：【严重/高危】XXXX系统存在XX漏洞漏洞地址：https://xxx.xxx.com/xxx漏洞类型：未授权访问 / 信息泄露 / 弱口令 / 路径穿越危害等级：P1严重 / P2高危厂商名称：XXX公司测试账号：无（匿名访问）测试时间：2026年XX月XX日
```  
### 二、漏洞原理  
```
由于系统对XX接口/路径未做严格身份鉴权，攻击者可在未登录情况下，直接访问/调用相关接口，导致敏感信息泄露/权限绕过/服务器文件读取等风险。
```  
### 三、复现步骤（必写）  
```
1. 访问URL：https://xxx.xxx.com/xxx2. 删除请求头中 Cookie、Token、Authorization3. 直接发送请求，返回200状态码4. 成功获取敏感数据/进入后台/读取系统文件5. 漏洞可稳定复现，无利用条件限制
```  
### 四、漏洞危害（决定分数高低）  
```
1. 可泄露用户姓名、手机号、身份证、地址、订单信息2. 可泄露管理员账号、数据库账号密码、云密钥3. 攻击者可进一步登录后台，操作业务数据4. 可获取服务器权限，读取配置文件、源码、数据库5. 漏洞影响范围广，可批量获取数据，危害极大
```  
### 五、修复建议  
```
1. 对接口添加严格登录鉴权、Token校验2. 禁止测试环境、后台地址直接暴露在公网3. 禁用默认口令/弱口令，启用强密码策略4. 限制敏感路径访问权限，关闭危险端点5. 对前端JS进行脱敏处理，删除密钥、内网地址
```
